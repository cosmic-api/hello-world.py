What is Cosmic?

Cosmic is a high-level framework for crafting web APIs. This page gives you an overview of how to write a simple Cosmic API. Instructions on how to run this example can be found on GitHub project.

The Server

from cosmic.tools import normalize_schema
from cosmic.api import API

# Define name of the API
api = API.create('cosmic_hello_world')

normalize_schema that we are importing here is a helper function that turns JSON-like data structure into a schema model. We'll get to see how it's used pretty soon.

API class is used to create new instance of Cosmic API.

# Set of people
people = set()

This is where we store bunch of names.

@api.action(
    accepts=normalize_schema({"type": "string"}),
    returns=normalize_schema({"type": "string"})
)

Here we are annotating remember_me function as an API endpoint. This function then automatically gets assigned an URL /actions/remember_me.

Since remember_me will take in string name of a person, we set accepts keyword argument to a schema which represents a single string type. Note that normalize_schema helper function constructs proper schema object from JSON-like data structure.

returns keyword argument defines return value of the remember_me function. Which is also simply a string.

def remember_me(name):
  """Remember names upto 10 people.
  """
  if name not in people:
    # Too many people to remember!
    if len(people) >= 10:
      people.pop()
    # Add new person
    people.add(name)
    return "Hello, %s! I'll remember you." % name
  else:
    return "Welcome back, %s." % name

Actual function that remembers up to 10 names.

Note that the parameter name is a regular python str type. What is happening under the hood is that the JSON body of HTTP request is parsed, validated, and normalized to python's native str type.

The return value goes through a similar process as well, returned native str type is validated and serialized into JSON, forming application/json response.

@api.action(
    returns=normalize_schema({
        "type": "array",
        "items": {"type": "string"}
    })
)
def list_people():
  """List everyone I know.
  """
  return list(people)

Second action, which takes no argument, returns list of names in the set.

Here we see a bit more complex return type. list_people function returns a python list, which will then be validated, and serialized to a JSON array containing strings. 

# Start listening to HTTP requests
api.run(port=8080, debug=True)

When the server is started, Cosmic generates an endpoint at /spec.json which describes all endpoints on this server precisely. We think that well defined, simple, and portable specs like these help both API developers and consumers to work together nicely.

Using the API with Cosmic Client

APIs built with Cosmic are well defined by the spec. This allows us to build native client libraries on the fly by loading /spec.json from the server.

For example, with hello-world.py, we can do

from cosmic.api import API
client = API.load('http://localhost:8080/spec.json')

client.actions.remember_me("Cosmic") # prints u"Hello, Cosmic! I'll remember you."
client.actions.remember_me("Cosmic") # prints u'Welcome back, Cosmic.'

client.actions.remember_me("Hello") # prints u"Hello, Hello! I'll remember you."

client.actions.list_people() # returns [u'Cosmic', u'Hello']
